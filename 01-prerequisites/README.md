## DESCRIPTION 

**End-to-End Monitoring Setup on AWS EKS using Prometheus & Grafana with IRSA and EBS CSI Driver**

<img width="1536" height="1024" alt="image" src="https://github.com/user-attachments/assets/e2e94e47-3ec3-4a32-b0f4-c3a59146218c" />


This project demonstrates setting up a complete Kubernetes monitoring stack on Amazon EKS using:

- Amazon EKS (eksctl)

- Metrics Server

- IRSA (IAM Roles for Service Accounts)

- EBS CSI Driver

- Prometheus (Helm)

- Grafana (Helm)

- Java Application Monitoring

The project includes real-world troubleshooting and production-grade setup practices.

# Key Learnings

Metrics API troubleshooting

Fixing metrics-server TLS issue

IRSA secure IAM implementation

Persistent storage using EBS

Helm-based deployment strategy


# Tools Used

a. Amazon Web Services (EKS)

b. Kubernetes

c. eksctl

d. Helm

e. Prometheus

f. Grafana

g. Metrics Server

h. IRSA (OIDC based IAM integration)
